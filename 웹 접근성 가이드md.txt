위 3개 케이스 다 보여줘

알겠어. 이건 실무에서 바로 써먹는 패키지로 정리해줄게.
아래 3가지는 감사 대응 기준으로 실제 통하는 수준이야.
(형식 그대로 써도 되고, 내부 문서/메일/보고서로 바로 옮겨도 됨)

① 우리 웹진이 감사에서 위험한 상태인지 체크리스트

👉 YES가 많을수록 위험
👉 🔴 3개 이상이면 “지적 가능성 높음”

🧩 기본 구조

⬜ 웹진이 별도 도메인이다

⬜ 웹진에 이벤트/참여형 페이지가 있다

⬜ 외부 콘텐츠(개인 블로그, SNS, 외부 이미지)를 직접 노출한다

♿ 웹 접근성

⬜ 웹진에 웹 접근성 품질마크가 없다

⬜ KWCAG 2.1 자체 점검 결과가 없다

⬜ 이미지 alt, 제목 구조, 키보드 접근 미확인

⬜ 외부 콘텐츠 접근성 보장 불가

👉 2개 이상이면 접근성 지적 가능

🔐 개인정보·보안

⬜ 이벤트/설문/구독 기능이 있다

⬜ 개인정보 처리방침이 웹진 도메인에 없다

⬜ 외부 솔루션(구글폼 등) 연계

⬜ 보안 점검/취약점 점검 이력 없음

👉 하나라도 YES면 감사 질문 대상

📸 저작권

⬜ 개인 블로그 글/사진 사용

⬜ 사용 동의서 없음

⬜ 출처 표시만 하고 사용

⬜ 직원 개인 콘텐츠 활용

👉 YES 1개면 바로 지적 가능

🧾 외주 책임 리스크

⬜ 계약/RFP에 “이벤트 진행·활성화 방안 제시”

⬜ “유지·보수 주관” 문구 있음

⬜ 외주사 권고·주의 메일 기록 없음

👉 YES면 외주사도 엮일 수 있음

🎯 판정 기준

🟢 0~2개 → 안정

🟡 3~5개 → 관리 필요

🔴 6개 이상 → 감사 지적 위험

② 품질마크 없이 감사 대응용 문구 & 근거 문서 예시

👉 핵심은 **“마크 없음 ≠ 미준수”**를 증명하는 것

📄 (1) 감사 대응용 공식 문구 (보고서/답변서)
본 웹진은 별도 도메인으로 운영되고 있으나,
한국형 웹콘텐츠 접근성 지침(KWCAG 2.1)을 기준으로
구조 설계 및 콘텐츠 제작을 진행하고 있으며,
정기적인 자체 점검을 통해 접근성 개선을 수행하고 있습니다.

현재 웹 접근성 품질마크는 미취득 상태이나,
향후 인증 범위 확대 또는 신규 인증 추진을 검토 중에 있습니다.


👉 이 문구 실제로 감사 답변에 쓰인다

📄 (2) 내부 근거 문서 제목 예시 (이게 중요)

아래 중 2개 이상 있으면 방어력 급상승

「웹진 웹 접근성 자체 점검 결과 보고서」

「KWCAG 2.1 항목별 준수 체크리스트」

「외부 콘텐츠 사용 시 접근성·저작권 검토 의견」

「웹진 유지보수 범위 및 책임 구분 문서」

📧 (3) 외주사/기관 방어용 메일 문구
웹진은 KWCAG 2.1 기준을 준수하여 기술적 유지보수를 수행 중이며,
콘텐츠 기획, 외부 자료 사용, 이벤트 운영에 대한
최종 책임은 발주처 소관으로 이해하고 있습니다.

외부 개인 콘텐츠 활용 시 접근성 및 저작권 관리가
어려울 수 있어 내부 검토 후 진행을 권고드립니다.


👉 이 메일 한 통만 있어도 책임 구분이 갈림

③ 웹진을 웹 접근성 품질마크 인증 범위에 포함시키는 실무 절차

👉 “나중에 포함할 예정”이라는 말이 빈말이 안 되게 만드는 프로세스

🧭 STEP 1. 구조 판단 (제일 중요)

아래 중 어디에 해당?

구조	난이도
메인 도메인 하위 경로 /webzine	⭐ 쉬움
서브도메인 webzine.xxx.go.kr	⭐⭐ 보통
완전 별도 도메인	⭐⭐⭐ 어려움
🧾 STEP 2. 인증 범위 변경 or 신규 인증 결정

기존 마크 있음 → 인증 범위 확대

기존 마크 없음 → 신규 인증

👉 인증기관에 사전 문의 필수

♿ STEP 3. 사전 접근성 점검 (필수)

자동 점검 + 수동 점검

이벤트 페이지 포함

외부 콘텐츠 제거 or 내부 전환

📌 이 단계 안 하면 100% 탈락

📑 STEP 4. 인증 신청

필요 자료:

사이트 개요

인증 범위 URL

접근성 점검 결과

개인정보처리방침

🧪 STEP 5. 심사 대응

이미지 alt

제목 구조

키보드 접근

오류 수정 이력

🏁 STEP 6. 마크 부착

인증된 도메인에만 부착

인증 범위 외 페이지 ❌ 사용 금지

🎯 실무 팁 (진짜 중요)

당장 인증 안 받아도
“사전 점검 + 인증 검토 기록”만 있어도
감사에서 태도가 완전히 달라진다

🔚 최종 한 줄 정리

별도 웹진 + 이벤트 + 외부 콘텐츠 + 품질마크 없음은
감사에서 위험하지만,
점검·문서·권고 기록이 있으면 충분히 방어 가능하다.
