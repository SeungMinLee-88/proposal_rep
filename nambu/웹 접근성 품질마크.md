웹 접근성 품질마크가 있는 홈페이지의 웹진이 별도 홈페이지로 존재하면 웹 접근성 품질마크 없어도 되냐?

간단하게 결론부터 말씀드리면,,,

➡️ 메인 홈페이지에 ‘웹 접근성 품질마크’가 있어도,
그 **별도 분리된 웹진(서브사이트/서브도메인/마이크로사이트 등)**이 독립적으로 운영되는 경우는
그 웹진까지 별도 인증을 받아야 마크를 표시할 수 있습니다.

즉, 웹진이 본체와 별도 서비스로 존재한다면 품질마크 없어도 된다 → ❌ 아님
→ 독자적인 웹사이트로 운영되는 경우 접근성 인증 범위에 포함되지 않으면 마크 표기는 적합하지 않습니다.

🧾 왜 그런가요?
✔ 웹 접근성 품질마크는 사이트 단위 인증

웹 접근성 품질마크는 지능정보화 기본법에 따라 웹사이트 전체가 접근성 기준(KWCAG 2.1) 준수 여부를 심사해 부여하는 인증입니다.

인증서에도 **‘인증범위: ○○ ○○홈페이지’**처럼 명시되며, URL 주소가 인증 대상입니다.

👉 인증은 “특정 도메인의 웹사이트(서비스)” 단위로 이뤄지고
서브 도메인 또는 분리된 웹진 도메인을 포함하지 않는 한, 웹진은 그 인증 범위에 들어가지 않아요.
(대부분 인증범위가 메인 도메인 한정으로 표시됨)

📌 예시 상황

만약 다음과 같다면…

✅ 메인 사이트:

https://company.kr   ← 품질마크 인증 O


별도 운영 웹진:

https://magazine.company.kr
https://company.kr/webzine


→ **서브 도메인(예: magazine.company.kr)**나
→ 메인 도메인 안이라도 별도 구조로 운영되는 웹진 페이지는
자동으로 접근성 인증 범위에 들어가지 않습니다

즉,
📍 “main site에 인증 마크 있으니까 별도 웹진은 면제” → 아님
📍 “웹진을 품질마크 범위로 인증받아 표시하려면 별도 심사 필요” → 맞음

📌 참고 기준

✔ 웹 접근성 품질마크는

해당 사이트 URL 전체가 심사·평가된 후 부여되고

인증서/마크 부착은 인증받은 주소/도메인에 한정되는 경우가 일반적입니다.

즉,
🔹 메인 도메인에만 인증된 상태라면
→ 메인 도메인 내 콘텐츠·페이지는 모두 인증 범위로 볼 수 있지만
→ 별도 서비스/사이트 구조는 포함되지 않습니다.

🧩 실무적으로 정리
경우	웹진 품질마크 필요 여부

웹진이 메인 내부 페이지 단순 링크 형태	✔ 인증 범위에 포함 가능성 높음

단, 실제 포함 여부는 품질마크 인증기관/심사 규정에 따라 다를 수 있으며,
애매한 경우 인증기관에 문의해 “웹진도 인증 범위에 포함되는지” 확인하는 것이 가장 확실합니다.

✨ 정리

✅ 웹 접근성 품질마크는 메인 홈페이지 단위 인증입니다.
✅ **별도 웹진(독립 서비스)**은 인증 대상이 아니면 마크 부착 불가입니다.
